# CAMPFLOW V1 - PostgreSQL Ready Checklist

Objectif: preparer la migration PostgreSQL sans la faire maintenant.

## Etat global

CAMPFLOW n'est pas encore compatible PostgreSQL directement. La logique SQL est centralisee, ce qui aide, mais elle depend de SQLite dans plusieurs zones.

## Ce qui est pret

- Les acces DB passent majoritairement par `database/db.py`.
- Le schema est explicite dans `database/schema.sql`.
- Les tables ont des identifiants entiers simples.
- Les donnees metier sont deja structurees par `establishment_id`.
- Les requetes sont dans `database/queries.py`, `database/auth.py`, `database/audit.py`.

## Ce qui cassera

| Zone | Probleme |
| --- | --- |
| Connexion | `sqlite3.connect`, `sqlite3.Row`, `PRAGMA`, `journal_mode`, `busy_timeout` |
| Placeholders | SQLite utilise `?`, PostgreSQL utilise `%s` avec psycopg/psycopg2 |
| Schema | `INTEGER PRIMARY KEY AUTOINCREMENT` doit devenir `SERIAL` ou `GENERATED BY DEFAULT AS IDENTITY` |
| Dates | Dates stockees en `TEXT`; PostgreSQL devrait utiliser `TIMESTAMP WITH TIME ZONE` ou `TIMESTAMP` |
| Pandas | `pd.read_sql_query` peut fonctionner, mais la connexion/cursor change |
| Migrations | Les migrations maison par `PRAGMA table_info` ne marcheront pas |
| Upserts | Certains `INSERT OR IGNORE` doivent devenir `ON CONFLICT DO NOTHING` |

## Fichiers a modifier

- `database/db.py`
- `database/schema.sql`
- `database/queries.py`
- `database/auth.py`
- `database/audit.py`
- `utils/backups.py`
- `utils/export_excel.py` si des chemins SQLite sont supposes
- scripts de maintenance dans `scripts/`

## Ordre exact conseille

1. Ajouter `DATABASE_URL` sans l'utiliser par defaut.
2. Creer une interface DB minimale dans `database/db.py`: `get_connection()`, row dict, transaction.
3. Extraire les differences SQL dans de petites fonctions utilitaires: placeholder, now SQL, insert ignore.
4. Ajouter un schema PostgreSQL separe.
5. Ecrire un script export SQLite vers CSV.
6. Ecrire un script import CSV vers PostgreSQL.
7. Tester en local avec PostgreSQL Docker.
8. Basculer Render staging vers PostgreSQL.
9. Faire une sauvegarde SQLite finale.
10. Basculer production.

## Estimation difficulte

- Couche connexion: moyenne.
- Schema: moyenne.
- Requetes: moyenne a haute, car nombreuses.
- Migration donnees: moyenne.
- Risque global: moyen si staging existe, eleve sans staging.

## Decision V1

Ne pas migrer dans cette passe. Pour une vraie exploitation au-dela d'un pilote, PostgreSQL devient prioritaire apres validation terrain.
